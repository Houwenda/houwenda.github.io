upload-labs

       复现了一下upload-labs，做一下整理，方便查阅。docker环境中的Linux环境和Windows环境差异较大，网上的Windows版本的很多writeup不适用于Linux版，这里整理的是Windows版，有空再研究下Linux版。

Pass-01

源码

题解

浏览器console里覆盖checkFile()函数，使其始终返回true
function checkFile() {return true;}

Pass-02

源码

题解

通过$_FILES['upload_file']['type']得到上传文件的MIME类型
抓包得到的请求消息体如下

修改其中Content-Type: application/octet-stream为Content-Type: image/png即可

Pass-03

源码

题解

$file_name = trim($_FILES['upload_file']['name']); 去除文件名前后的NULL、制表符、换行符、垂直制表符、回车、空格
$file_name = deldot($file_name);//删除文件名末尾的点deldot并非PHP内置函数
$file_ext = strrchr($file_name, '.'); 提取 . 后的文件拓展名
$file_ext = strtolower($file_ext); //转换为小写 将拓展名转换为小写（无法大小写绕过）
$file_ext = str_ireplace('::$DATA', '', $file_ext);//去除字符串::$DATA PHP在Wndows中如果文件名+”::$DATA”会把::$DATA之后的数据当成文件流处理,不会检测后缀名.且保持”::$DATA”之前的文件名
$file_ext = trim($file_ext); //收尾去空

如果没有对文件名格式进行修改，黑名单过滤中没有禁止上传.htaccess，可以尝试上传分布式配置文件.htaccess，将jpg拓展名的文件交给PHP处理

前提：开启mod_rewrite模块， AllowOverride All

但是$img_path = UPLOAD_PATH.'/'.date("YmdHis").rand(1000,9999).$file_ext; 对文件名进行了随机处理，上传文件后得到的文件名形如201905161335333060.htaccess，而需要的文件名是.htaccess，此方法无效

黑名单中只过滤了普通PHP等cgi文件拓展名，因此可以尝试其他可能被解析的拓展名

PHP ：php3 php4 php5 phtml pht
ASP ：asa cer cdx
aspx ：ashx asmx ascx
JSP ：jspx jspf

这里是PHP环境，因此尝试php3 php4 php5 phtml pht，其中phtml被成功解析执行

Pass-04

源码

题解

跟Pass-03相比增加了黑名单条目，不常用后缀名也被禁止上传，.htaccess未被过滤但无法利用

当上传名为shell.php:.jpg时，upload目录下会产生一个空的shell.php文件。

对于windows环境的服务器，上传1.asp:.jpg类型的文件，当文件传到服务端时，windows会将该文件识别成ADS，从而认为其宿主文件名为1.asp而将.jpg识别为流名。在测试过程中（测试环境中WEB Server为IIS、Apache、Nginx）无论如何构造都无法访问可以解析的文件。 Windows中冒号截断的原理

在upload目录下，默认只有shell.php可见，cmd执行 dir /a /r 可以看到shell.php和shell.php:.jpg:$DATA两个文件，删除shell.php时ntfs流也会被删除。

产生这一漏洞的PHP代码特点不是很清楚，与此题相似的前后几题都无法利用此方法。

在Windows中PHP在处理正则匹配时会有如下特性：

  • 双引号 = 点号
  • 大于号 = 问号
  • 小于号 = 星号

利用通配符shell.< shell.<<< shell.>>> shell.>><等可以向shell.php其中写入任意代码

以此类推，也可以通过相同的方式上传.htaccess文件，使jpg文件被PHP解析：
先上传jpg拓展名的PHP shell，获得随机文件名，再上传.htaccess文件，其中第一次文件名为.htaccess:.jpg，第二次为.htaccess，并传入内容。

源码

题解

缺少了统一大小写的步骤，可以用大小写绕过检验

使用文件拓展名.phP即可绕过

Pass-06

源码

题解

黑名单过滤了.htaccess，但是没有使用trim()函数去除文件名前后的NULL、制表符、换行符、垂直制表符、回车、空格

文件名后加上一个空格即可绕过过滤

源码

题解

缺少去除末尾.的操作deldot()

文件名后加一个.即可

Pass-08

源码

题解

没有对::$DATA进行过滤，Windows处理时将其当做NTFS备用数据流处理，上传shell.php::$DATA即可

Pass-09

源码

题解

没有使用随机文件名，文件名可控，且会删除末尾的点，文件名为 shell.php. . 即可

Pass-10

源码

题解

$file_name = str_ireplace($deny_ext,"", $file_name); 将文件名中的不合法拓展名改为空，可以通过双写绕过，文件名用shell.pphphp即可

Pass-11

源码

题解

上传路径从GET参数传入，与文件名拼接后保存。可以使用%00截断（CVE-2006-7243），字符串拼接时%00后的字符被截断。GET参数为save_path=../upload/shell.php%00，文件名为shell.jpg即可

Pass-12

源码

题解

保存文件目录改成了从POST参数传入，依然存在00截断漏洞，在POST消息体里添加十六进制0x00即可

Pass-13

源码

题解

定义了一个函数getReailFileType($filename)，通过文件头识别文件类型。没有过滤拓展名，找张图片写入shell，文件包含即可

Pass-14

源码

题解

定义了isImage($filename)函数，用getimagesize和image_type_to_extension函数提取文件信息，判断合法文件类型，在POST消息体的shell前添加GIF89a，文件被保存为gif拓展名，文件包含利用即可

Pass-15

源码

题解

定义isImage($filename)方法，用exif_imagetype判断合法文件类型。方法同Pass-14

源码

二次渲染，需要构造图片文件，使处理后的文件含有可控shell
upload-labs之pass 16详细分析

上传的文件未经过滤，但会在改名后删除原文件，可以利用条件竞争，连续写入shell.php
